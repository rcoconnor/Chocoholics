Use Case: Receive Health Care Service

Context: A customer approaches the provider to receive health care services.

Actors: The Chochoholic, The Provider.

Main Success Scenario:

1. The Chocoholic hands their member card to the provider
  1.1 The Provider swipes The Chocoholic member card
2. The card number is sent to the ChocAn data center
3. The data center checks the number against the member database
4. The data center returns that the number is valid
5. The provider hands back the card, and wishes the customer a good day

Extensions:

1.1a. The Card number must be entered manually into the computer
4a. The member number is invalid because that member does not exist, so the data center return invalid number
4a. The member number is invalid because that member has been suspended, so the data center returns "member suspended" (Potential for overdue fees as well)

Use Case: Bill for Health Care Services

Context: The customer has recieved a service, and the provider bill ChocAn for their time/resources

Actors: The Chocoholic, The Provider, Disk Drive

Main Success Scenario:

1. The Customer hands his card to the provider
  1.1 The Provider swipes their card
2. The card number is sent to the data center
  2.1 The card number is checked for validity
  2.2 The data center sends that the card is valid back
3. The Provider looks up the six digit service code
4. The Provider enters the code into the program
5. That program displays the name of the service associated and asks the provider to verify it
6. The Provider verifies the service
7. The Provider is offered the oppurtunity to write commentary about the service provided
8. The program writes a file to the hard drive including:
    Current Date and Time (MM/DD/YYYY HH:MM:SS)
    Date Service was Provided (MM/DD/YYYY)
    Provider Number (9 digits)
    Member Number (9 digits)
    Service Code (6 digits)
    Comments (100 characters) (optional)
9. The program looks up the fees for the service provided from the provider directory
  9.1 The fee is displayed on the provider's terminal
10. The Provider verifies the fees
  10.1. The Provider fills out a form with the current date and time, the date the service was provided, member name and number, service code, and fee to be paid, that is saved on the terminal
11. The Provider hands the card back to the Chocoholic

Extensions:

3a. The Provider requests a full copy of the provider directory
  3a.1. The chocAn center sends the directory as an email attachment
5a. The code entered is incorrectly formatted or wrong, and the program displays an error and returns to the enter code screen
6a. The Provider does not verify the code, because it is incorrect, and the program returns to the enter service code line
